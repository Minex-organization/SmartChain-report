En esta seccion se detallaran cada User Stories para divididas por las epicas establecidas en la sección anterior:

| **Epic / Story ID** | **Title** | **Description** | **Acceptance Criteria** | **Related with (Epic ID)** |

| **EP01** | **Extracción Minera y Operaciones Offline** | **Gestión del registro inicial de lotes en mina, operatividad sin conexión y sincronización de datos.** | | |
| **US01** | Registro de lote de extracción | Como Operador Minero, quiero registrar un nuevo lote de minerales extraídos en la plataforma para que su origen y peso queden documentados desde el inicio. | *Escenario 1: Registro exitoso de un nuevo lote.* <br><br>**Dado que** el operador minero requiere registrar un nuevo lote, <br>**Cuando** envíe los detalles válidos del mineral (tipo, peso bruto, fecha de extracción), <br>**Entonces** el sistema registrará el lote y generará un ID único de trazabilidad. <br><br>*Escenario 2: Validación de campos obligatorios faltantes.* <br><br>**Dado que** el operador minero procesa el registro, <br>**Cuando** omita ingresar datos obligatorios como el peso bruto o el tipo de mineral, <br>**Entonces** el sistema rechazará la solicitud y requerirá que se completen los campos faltantes. <br><br>*Escenario 3: Validación de peso negativo o inválido.* <br><br>**Dado que** el operador minero procesa el registro, <br>**Cuando** envíe un peso de mineral menor o igual a cero, <br>**Entonces** el sistema invalidará la operación indicando que el peso debe ser mayor a cero. | EP01 |
| **US02** | Ingreso de datos offline | Como Operador Minero, quiero registrar datos sin conexión en la plataforma para que mi trabajo no se interrumpa cuando me encuentro en zonas remotas de la mina. | *Escenario 1: Guardado local sin conexión.* <br><br>**Dado que** el dispositivo del operador no tiene conexión a la red, <br>**Cuando** confirme el registro de un lote, <br>**Entonces** el sistema persistirá los datos en el almacenamiento local del dispositivo. <br><br>*Escenario 2: Activación de modo offline.* <br><br>**Dado que** el operador está utilizando el sistema, <br>**Cuando** el dispositivo pierda la señal de red, <br>**Entonces** el sistema activará internamente el modo offline y advertirá al usuario sobre el cambio de estado de conexión. <br><br>*Escenario 3: Restricción de consultas en tiempo real.* <br><br>**Dado que** el sistema se encuentra en modo offline, <br>**Cuando** el operador requiera acceder a información que depende de sincronización en tiempo real, <br>**Entonces** el sistema denegará la consulta indicando la falta de conectividad. | EP01 |
| **US03** | Sincronización automática de datos offline | Como Operador Minero, quiero que la aplicación sincronice mis datos locales automáticamente al recuperar la conexión para que la base de datos central se actualice. | *Escenario 1: Sincronización exitosa.* <br><br>**Dado que** existen lotes guardados en el almacenamiento local, <br>**Cuando** el dispositivo recupere la conexión a la red, <br>**Entonces** el sistema iniciará automáticamente el envío de los registros al servidor central. <br><br>*Escenario 2: Manejo de conflictos de sincronización.* <br><br>**Dado que** el sistema inicia la sincronización de datos locales, <br>**Cuando** detecte un conflicto con registros existentes en el servidor, <br>**Entonces** el sistema registrará la anomalía y preservará la versión validada por el servidor como fuente de verdad. <br><br>*Escenario 3: Recuperación ante fallo de envío.* <br><br>**Dado que** la sincronización automática está en proceso, <br>**Cuando** la conexión se interrumpa nuevamente, <br>**Entonces** el sistema pausará el envío y mantendrá los datos locales intactos para el próximo intento de conexión. | EP01 |
| **US04** | Reporte de anomalías en extracción | Como Supervisor, quiero reportar anomalías o pérdidas durante la extracción para mantener una auditoría clara sobre las fugas de producción en mina. | *Escenario 1: Registro de anomalía.* <br><br>**Dado** un lote previamente registrado en el sistema, <br>**Cuando** el supervisor emita un reporte de anomalía con su respectiva justificación descriptiva, <br>**Entonces** el sistema actualizará el estado del lote a "Alerta" y registrará la marca de tiempo. <br><br>*Escenario 2: Adjuntar evidencia visual.* <br><br>**Dado que** el supervisor emite un reporte de anomalía, <br>**Cuando** envíe evidencia fotográfica en formatos permitidos, <br>**Entonces** el sistema procesará y vinculará los archivos multimedia al registro de la anomalía. | EP01 |
| **US05** | Impresión de código QR del lote | Como Encargado de Logística, quiero imprimir un código QR para cada lote registrado para que pueda ser adherido físicamente a los sacos de mineral. | *Escenario 1: Generación de QR.* <br><br>**Dado** un lote registrado y sincronizado exitosamente con el servidor central, <br>**Cuando** el encargado logístico solicite la generación de su etiqueta de identificación, <br>**Entonces** el sistema compilará y proveerá un documento con el código QR y los datos resumidos del mineral. <br><br>*Escenario 2: Restricción por falta de sincronización.* <br><br>**Dado** un lote que se encuentra únicamente en almacenamiento local (modo offline), <br>**Cuando** se solicite la generación de su etiqueta QR, <br>**Entonces** el sistema denegará la acción exigiendo la sincronización previa con el servidor central. | EP01 |

| **EP02** | **Procesamiento en Refinería y Logística** | **Control de transporte, custodia legal con firma digital y procesos de recepción o división de lotes.** | | |

| **US07** | Recepción de lote en refinería | Como Operario de Refinería, quiero confirmar la llegada de un lote escaneando su QR para que la mina sepa que el material llegó a destino. | *Escenario 1: Confirmación de peso.* <br><br>**Dado** un lote con estado "En Tránsito", <br>**Cuando** el operario envíe los datos de validación incluyendo el peso de entrada, <br>**Entonces** el sistema actualizará el estado del lote a "Recibido en Refinería". <br><br>*Escenario 2: Alerta por merma.* <br><br>**Dado** el registro de recepción de un lote, <br>**Cuando** el peso ingresado sea menor al 95% del peso de extracción original, <br>**Entonces** el sistema registrará automáticamente una "Alerta de Merma". <br><br>*Escenario 3: Notificación.* <br><br>**Dado** el registro de un lote recibido exitosamente, <br>**Cuando** se actualice el estado en la base de datos, <br>**Entonces** el sistema emitirá una notificación de confirmación de entrega al origen. <br><br>*Escenario 4: Evidencia de sellos.* <br><br>**Dado** un lote con sellos de seguridad reportados como alterados, <br>**Cuando** el operario procese la recepción, <br>**Entonces** el sistema exigirá el envío de evidencia fotográfica obligatoria para permitir el registro. | EP02 |
| **US08** | División de lotes procesados (Split) | Como Operario de Refinería, quiero dividir un lote bruto en unidades más pequeñas para distribuirlas a joyerías manteniendo la trazabilidad. | *Escenario 1: División exitosa.*<br><br>**Dado** un lote procesado en refinería, <br>**Cuando** el operario envíe una solicitud de división especificando el número de partes y sus pesos, <br>**Entonces** el sistema generará nuevos identificadores (lotes hijos) vinculados estructuralmente al lote original. <br><br>*Escenario 2: Peso excedente.* <br><br>**Dado** el proceso de división de un lote, <br>**Cuando** la suma del peso de las fracciones solicitadas exceda el peso total del lote original, <br>**Entonces** el sistema denegará la operación por inconsistencia de datos. <br><br>*Escenario 3: Etiquetado masivo.* <br><br>**Dado** un conjunto de lotes hijos recién generados, <br>**Cuando** se solicite su identificación física, <br>**Entonces** el sistema compilará y proveerá los documentos con los códigos QR respectivos para su impresión. <br><br>*Escenario 4: Herencia de historial.* <br><br>**Dado** un lote hijo resultante de una división, <br>**Cuando** se consulte su historial de trazabilidad, <br>**Entonces** el sistema proveerá la información heredada detallando su origen exacto en el lote padre y la mina de procedencia. | EP02 |
