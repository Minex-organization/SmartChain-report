# CAPÍTULO III: REQUIREMENTS SPECIFICATION

## 3.1. Epics

En esta sección se describen las Épicas identificadas para el sistema, las cuales agrupan las funcionalidades clave del negocio.

ID Épica | Nombre de la Épica | Descripción General | Cantidad de HUs |

| US03 | Sincronización automática de datos offline | Como Operador Minero, quiero que la aplicación sincronice mis datos locales automáticamente al recuperar la conexión para que la base de datos central se actualice. | *Escenario 1: Sincronización exitosa.* <br><br>**Dado que** el operador tiene lotes guardados localmente, <br>**Cuando** el dispositivo recupere una conexión estable, <br>**Entonces** el sistema enviará los registros al servidor central. <br><br>*Escenario 2: Notificación de sincronización.* <br><br>**Dado que** la aplicación ha iniciado la sincronización, <br>**Cuando** todos los registros se hayan subido, <br>**Entonces** el sistema emitirá una notificación indicando "Datos offline sincronizados correctamente". <br><br>*Escenario 3: Manejo de conflictos.* <br><br>**Dado un** lote offline con un ID que ya existe en la nube, <br>**Cuando** se sincronice, <br>**Entonces** el sistema le asignará un nuevo ID único y notificará al usuario. <br><br>*Escenario 4: Recuperación de fallo.* <br><br>**Dado que** el sistema está sincronizando, <br>**Cuando** la red se caiga nuevamente, <br>**Entonces** el sistema pausará la subida y la retomará desde el punto exacto al volver la conexión. | EP01 |
